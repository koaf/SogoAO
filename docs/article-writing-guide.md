# WABI-LAB 年内受験対策 記事作成ガイド

このドキュメントは、総合型選抜・学校推薦型選抜の記事を ChatGPT で安定して作るための書き方ルールです。既存記事の構成、文体、SEO設計、Markdown形式を基準にしています。

## メディア方針

- 対象読者は高校生、保護者、高校教員、年内入試を検討している受験生。
- テーマは総合型選抜、学校推薦型選抜、志望理由書、小論文、面接、評定平均、出願スケジュール、保護者の関わり方。
- 目的は不安をあおることではなく、制度理解と具体的な準備行動につなげること。
- 読後に「何をすればよいか」が明確になる実用記事にする。

## 記事の基本トーン

- やさしく、断定しすぎず、でも曖昧に逃げない。
- 受験生目線の悩みから入り、保護者にも伝わる言葉で説明する。
- 専門用語は初出で必ず短く説明する。
- 「すごい実績がないと無理」ではなく、「経験をどう言語化するか」を重視する。
- 抽象論だけで終わらせず、表、チェックリスト、NG例とOK例、ステップを入れる。

## 文章スタイル

- Markdownで書く。
- 見出しは `##`、小見出しは `###`、必要に応じて `####` を使う。
- 1文は長くしすぎない。目安は40〜70字。
- 重要語は `**太字**` で強調する。
- 読者への注意喚起は引用ブロックを使う。
- 絵文字は既存記事に合わせて少量なら可。ただし使いすぎない。
- 最後は行動を促すまとめで締める。

## 記事の推奨構成

### 1. フロントマター

各記事は次の形式で始める。

```yaml
---
title: "記事タイトル"
date: YYYY-MM-DD
lastmod: YYYY-MM-DD
draft: false
description: "検索結果に出す説明文。80〜140字程度。"
summary: "記事一覧に出す短い要約。80〜140字程度。"
keywords: ["総合型選抜", "学校推薦型選抜", "志望理由書"]
categories: ["カテゴリ名"]
tags: ["総合型選抜", "志望理由書", "面接対策"]
authors: ["WABI-LAB"]
showTableOfContents: true
showReadingTime: true
showWordCount: true
showDate: true
showDateUpdated: true
showTaxonomies: true
showBreadcrumbs: true
---
```

### 2. はじめに

読者の悩みや誤解から入る。

例:

```markdown
## 【はじめに】

### 「何から始めればいいかわからない」人へ

総合型選抜や学校推薦型選抜は、一般選抜とは準備の進め方が大きく異なります。

この記事では、制度の基本から具体的な対策まで、初めての人にもわかるように整理します。
```

### 3. この記事でわかること

冒頭付近に箇条書きを入れる。

```markdown
### この記事でわかること

- 制度の基本
- 出願までのスケジュール
- 志望理由書・小論文・面接の対策
- よくある失敗と回避策
```

### 4. 本文

本文は大きく5〜7章に分ける。各章は次のいずれかを含めると読みやすい。

- 比較表
- スケジュール表
- NG例とOK例
- チェックリスト
- 3ステップ解説
- よくある質問

既存記事でよく使う章立て:

```markdown
## 1. まず基本を理解する
## 2. メリット・デメリット
## 3. 向いている人・注意が必要な人
## 4. スケジュールと準備の流れ
## 5. 書類・小論文・面接の対策
## 6. よくある失敗
## 【まとめ】今日からやること
```

## よく使うパーツ

### 比較表

```markdown
| 項目 | 総合型選抜 | 学校推薦型選抜 | 一般選抜 |
|------|------------|----------------|----------|
| 選考の中心 | 書類・面接・小論文 | 推薦・評定・書類 | 学力試験 |
| 出願条件 | 大学により異なる | 学校長推薦が必要 | 原則自由 |
```

### NG例とOK例

```markdown
| NG | OK |
|----|----|
| 「貴学は有名だから」 | 「〇〇教授の△△研究に関心があり、□□を学びたい」 |
```

### チェックリスト

```markdown
- [ ] 志望校のアドミッション・ポリシーを読んだ
- [ ] 出願条件と締切を確認した
- [ ] 志望理由を自分の経験と結びつけた
```

### 注意ブロック

```markdown
> 注意：募集要項は年度によって変わります。必ず大学公式サイトで最新情報を確認してください。
```

## タイトルの作り方

タイトルは検索意図と読者の悩みがわかる形にする。

良い例:

- `【完全版】総合型選抜とは？メリット・デメリットから対策までわかりやすく解説`
- `【例文あり】総合型選抜の志望理由書の書き方完全ガイド`
- `親が知っておくべき総合型選抜の特殊性と正しい見守り方`
- `学校推薦型選抜とは？指定校推薦・公募推薦の違いと準備スケジュール`

避ける例:

- `総合型選抜について`
- `受験対策まとめ`
- `志望理由書の書き方`

## SEO設計

### description

- 80〜140字程度。
- 記事で扱うテーマ、対象読者、得られる内容を入れる。
- キーワードを自然に含める。

例:

```yaml
description: "学校推薦型選抜の仕組み、指定校推薦・公募推薦の違い、評定平均、出願条件、面接・小論文対策まで、受験生と保護者向けにわかりやすく解説します。"
```

### keywords / tags

- メインキーワードを先頭に置く。
- 関連語を5〜10個程度入れる。
- tagsは記事一覧で表示されるため、多すぎないようにする。

## 記事の品質チェック

公開前に確認する。

- [ ] 読者の悩みから始まっているか
- [ ] 制度説明だけでなく、具体的な行動に落ちているか
- [ ] 表やチェックリストが1つ以上あるか
- [ ] 「大学によって異なる」内容は募集要項確認を促しているか
- [ ] 志望理由書や面接では、具体性と原体験を重視しているか
- [ ] 最後に今日からできる行動が示されているか
- [ ] フロントマターが正しい形式になっているか

## ChatGPT用プロンプト

新しい記事を書くときは、以下を ChatGPT に貼り付ける。

```text
あなたは「WABI-LAB 年内受験対策」の編集者です。
総合型選抜・学校推薦型選抜を受ける高校生と保護者向けに、SEOを意識した実用的なMarkdown記事を書いてください。

条件:
- テーマ: 【ここにテーマを入れる】
- 想定読者: 【高校生 / 保護者 / 高1・高2 / 高3 など】
- 文字量: 6000〜10000字程度
- 文体: やさしく、具体的で、実践的
- Markdown形式
- 冒頭にHugo/Vite用のフロントマターを付ける
- authorsは ["WABI-LAB"]
- 章立ては5〜7章
- 表、チェックリスト、NG例とOK例を少なくとも1つずつ入れる
- 専門用語は初出で説明する
- 最後は「今日からやること」がわかるまとめにする
- 募集要項や制度は大学・年度で変わるため、必要な箇所で公式情報確認を促す

記事構成:
1. 【はじめに】読者の悩みや誤解から入る
2. この記事でわかること
3. 基本知識
4. 具体的な準備・対策
5. よくある失敗や注意点
6. チェックリストまたはスケジュール
7. 【まとめ】今日からやること

出力はMarkdown本文だけにしてください。
```

## 新規記事ファイルの作り方

1. `content/posts/<slug>/index.md` を作る。
2. `<slug>` は英小文字とハイフンで書く。例: `gakkou-suisen-guide`。
3. 上記プロンプトで作ったMarkdownを貼る。
4. `npm run build` で表示確認する。
